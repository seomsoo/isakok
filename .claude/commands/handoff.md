현재 작업 상태를 docs/STATUS.md에 업데이트해줘.

## ⚠️ 절대 규칙 — 이전 단계 기록 보존

**기존 STATUS.md를 먼저 Read한 뒤 갱신한다.** 다음을 절대 압축·삭제하지 말 것:

- "완료된 것" 섹션의 이전 단계(1~N-1단계) 하위 항목 전부 — 컴포넌트명, 훅명, 서비스명, 마이그레이션 번호, Codex 수정 이력, 디자인 결정 등 모든 디테일
- "실패한 접근 (반복 금지)" 섹션의 모든 누적 항목 — 한 번 추가된 항목은 영구 보존

갱신 방식:
1. 이전 STATUS.md를 Read로 읽는다
2. **이전 단계 블록은 그대로 복사** (디테일 손대지 않음)
3. 이번 세션에서 한 작업을 새 단계 블록으로 **추가**
4. "현재 단계", "다음 할 것", "진행 중인 것"만 갱신
5. "실패한 접근"은 누적 (이전 항목 보존 + 이번에 새로 배운 것 추가)

> Why: 사용자가 다음 세션 진입 시 이전 단계 디테일을 STATUS.md로 빠르게 회상함. 압축하면 git log/blame을 일일이 따라가야 함. 이미 같은 실수가 4단계·5단계 두 번 반복됨.

## 기록할 내용

1. **현재 단계**: 몇 단계 작업 중인지
2. **상태**: 진행중 / 구현완료-검증대기 / 검증완료 / 다음단계대기
3. **완료된 것**: 이전 단계 블록 전부 보존 + 이번 세션 신규 블록 추가
4. **진행 중인 것**: 아직 끝나지 않은 작업 (있으면)
5. **다음 할 것**: 구체적인 다음 액션 1~3개
6. **알려진 문제**: 발견했지만 아직 안 고친 것 (있으면)
7. **실패한 접근**: 누적 — 이전 항목 보존 + 이번 세션에서 새로 배운 것만 추가

## 형식

docs/STATUS.md 갱신 시 아래 골격을 유지하되, "## 완료된 것" 안에는 이전 단계 디테일을 그대로 보존하고 이번 단계만 새로 덧붙인다:

```md
# 프로젝트 상태

> 마지막 업데이트: {오늘 날짜}

## 현재 단계

{N}단계: {단계명} — {상태}

## 완료된 것

- {파일 경로}: {한 줄 설명}

## 진행 중인 것

- {내용} (없으면 "없음")

## 다음 할 것

1. {가장 먼저 할 것}
2. {그 다음}

## 알려진 문제

- {내용} (없으면 "없음")

## 실패한 접근 (반복 금지)

- {내용} (없으면 "없음")
```

# 디자인 스타일 가이드

> 톤: 토스 스타일 (깔끔, 미니멀, 정보 중심)
> 컬러: Teal 기반 7색 시스템 유지
> 대상: 모바일 퍼스트 (375px 기준, max-width 430px)
> 최종 수정: 2026-04-02

---

## 1. 디자인 원칙

### 1-1. 여백이 정보다

- 콘텐츠 사이 여백을 넉넉히 확보한다
- 빈 공간을 채우려 하지 않는다 — 여백 자체가 시각적 계층을 만든다
- 요소 간 간격이 좁으면 관계가 있다는 뜻, 넓으면 구분이라는 뜻

### 1-2. 정보 계층을 명확히

- 한 화면에서 가장 중요한 정보 1개가 즉시 눈에 들어와야 한다
- 제목 → 보조 텍스트 → 액션 순서로 시선이 흐른다
- 부차적 정보는 작게, 연하게, 또는 접어둔다

### 1-3. 장식을 최소화

- 일러스트, 그라데이션, 패턴 배경 사용하지 않는다
- 아이콘은 의미 전달 목적일 때만 사용한다 (장식용 아이콘 금지)
- 보더, 그림자, 구분선은 꼭 필요한 곳에만 — 여백으로 구분이 되면 생략

### 1-4. 카드 기반 레이아웃

- 관련 정보를 카드로 묶어 스캔하기 쉽게 한다
- 카드 안에서의 정보 배치도 일관된 패턴을 유지한다
- 카드는 평평하게 — 그림자는 최소한으로

### 1-5. 텍스트가 UI다

- 토스의 핵심: 큰 글씨로 핵심 메시지를 먼저 보여준다
- 버튼 라벨, 안내 문구 하나하나가 UX — 문장은 짧고 명확하게
- "~해주세요" 대신 "~하기", "~할게요" 같은 간결한 톤

## 2. 컬러 시스템

### 2-1. 팔레트 (OKLCH)

컬러 포맷은 OKLCH를 사용한다 (지각 균일, 파생색 L값 조절만으로 생성, Tailwind v4 네이티브 지원).

| 토큰        | OKLCH                     | HEX 참고 | 용도                              |
| ----------- | ------------------------- | -------- | --------------------------------- |
| `primary`   | `oklch(0.60 0.104 184.7)` | #0D9488  | CTA 버튼, 선택 상태, 진행률, 링크 |
| `secondary` | `oklch(0.33 0.030 285)`   | #333344  | 본문 텍스트, 헤딩                 |
| `tertiary`  | `oklch(0.95 0.019 193)`   | #E0F2F1  | 선택된 카드 배경, 태그, 뱃지      |
| `neutral`   | `oklch(0.98 0.003 85)`    | #F8F7F5  | 페이지 배경                       |
| `warning`   | `oklch(0.70 0.187 48)`    | #F97316  | 밀린 할 일, 주의 뱃지             |
| `critical`  | `oklch(0.64 0.208 25)`    | #EF4444  | 필수 뱃지, 에러, 삭제             |
| `success`   | `oklch(0.70 0.149 163)`   | #10B981  | 완료 체크, 성공 토스트            |

### 2-2. 확장 색상

| 토큰           | OKLCH                   | HEX 참고 | 설명                       |
| -------------- | ----------------------- | -------- | -------------------------- |
| `muted`        | `oklch(0.53 0.025 286)` | #6B6B7B  | 보조 텍스트, caption       |
| `placeholder`  | `oklch(0.70 0 0)`       | #9E9E9E  | placeholder, 비활성 텍스트 |
| `border`       | `oklch(0.95 0.003 85)`  | #F0EFED  | 구분선, 카드 외곽          |
| `border-input` | `oklch(0.91 0 0)`       | #E0E0E0  | 입력 필드 보더             |
| `surface`      | `oklch(1.00 0 0)`       | #FFFFFF  | 카드 배경                  |

### 2-3. 파생색 (L값 조절)

OKLCH의 핵심 장점: Lightness만 조절하면 같은 색조의 밝기 변형이 만들어진다.

```css
/* primary 파생색 — C(채도)와 H(색상)는 고정, L만 변경 */
--color-primary: oklch(0.6 0.104 184.7); /* 기본 */
--color-primary-hover: oklch(0.55 0.104 184.7); /* L -0.05 */
--color-primary-pressed: oklch(0.5 0.104 184.7); /* L -0.10 */
```

무채색(placeholder, border-input, surface)은 C=0이므로 L값만으로 밝기 스케일이 된다.

### 2-4. 컬러 사용 원칙

- **배경은 거의 흰색/회색**: 페이지 neutral, 카드 surface
- **포인트 컬러는 아낀다**: primary는 CTA 버튼, 선택 상태, 진행률 바에만 사용
- **텍스트로 계층 구분**: secondary(본문) → muted(보조) → placeholder(비활성)
- **시스템 컬러는 의미 있을 때만**: warning/critical/success는 상태 전달 목적에만
- **primary 배경에 흰 텍스트**: CTA 버튼 등
- **tertiary 배경에 primary 텍스트**: 태그, 칩, 선택 상태

## 3. 타이포그래피

### 3-1. 폰트

- 시스템 폰트 스택 사용 (웹폰트 로딩 없이 네이티브 느낌)
- `font-family: -apple-system, BlinkMacSystemFont, 'Apple SD Gothic Neo', 'Pretendard Variable', Pretendard, sans-serif`
- letter-spacing: `-0.01em` (전역, 한글 가독성)

### 3-2. 타입 스케일

| 레벨        | 사이즈 | Weight | Line Height | 용도                                         |
| ----------- | ------ | ------ | ----------- | -------------------------------------------- |
| **h1**      | 26px   | 700    | 1.35        | 페이지 메인 질문 ("이사 예정일이 언제예요?") |
| **h2**      | 22px   | 700    | 1.35        | 섹션 제목 ("오늘 할 일")                     |
| **h3**      | 18px   | 600    | 1.4         | 카드 제목, 서브 헤딩                         |
| **body**    | 16px   | 400    | 1.6         | 본문 텍스트, 리스트 항목                     |
| **body-sm** | 14px   | 400    | 1.5         | 보조 설명, 카드 내 부연                      |
| **caption** | 12px   | 400    | 1.5         | 날짜, 뱃지 텍스트, 하단 안내                 |

### 3-3. 타이포 규칙

- **h1은 화면당 1개**: 가장 중요한 메시지만
- **bold(700)은 아껴 쓴다**: h1, h2, 강조할 숫자에만
- **semibold(600)은 카드 제목 레벨**: h3, 버튼 라벨
- **본문은 항상 regular(400)**: body, body-sm, caption

## 4. 간격 체계

### 4-1. 기본 단위: 4px

모든 간격은 4px 배수로 구성한다.

| 토큰  | 값   | 용도                                   |
| ----- | ---- | -------------------------------------- |
| `xs`  | 4px  | 아이콘-텍스트 사이, 뱃지 내부 패딩     |
| `sm`  | 8px  | 같은 그룹 내 요소 간격, 칩 간격        |
| `md`  | 12px | 카드 간 간격, 리스트 아이템 간격       |
| `lg`  | 16px | 카드 내부 패딩, 섹션 내 요소 간격      |
| `xl`  | 20px | 페이지 좌우 패딩                       |
| `2xl` | 24px | 섹션 간 간격                           |
| `3xl` | 32px | 페이지 상단 여백, 큰 섹션 구분         |
| `4xl` | 48px | 페이지 하단 여백 (하단 버튼 영역 확보) |

## 5. 둥글기 / 그림자 / 보더

### 5-1. Border Radius

| 토큰          | 값     | 용도                             |
| ------------- | ------ | -------------------------------- |
| `radius-sm`   | 8px    | 입력 필드, 칩, 태그, 작은 버튼   |
| `radius-md`   | 12px   | 일반 버튼, 토스트, 드롭다운      |
| `radius-lg`   | 16px   | 카드, 모달, 바텀시트             |
| `radius-full` | 9999px | 원형 뱃지, 아바타, 프로그레스 바 |

### 5-2. 그림자

그림자는 최소한으로. 토스 스타일은 **그림자 대신 배경색 차이**로 깊이를 표현한다.

| 레벨        | 값                               | 용도                   |
| ----------- | -------------------------------- | ---------------------- |
| `shadow-sm` | `0 1px 2px rgba(0, 0, 0, 0.04)`  | 카드 (미세한 입체감만) |
| `shadow-md` | `0 4px 12px rgba(0, 0, 0, 0.08)` | 모달, 바텀시트, 토스트 |
| 없음        | -                                | 대부분의 요소          |

- 카드: `shadow-sm` 또는 그림자 없이 `border` 처리도 허용
- 플로팅 요소(모달, 바텀시트)만 `shadow-md`
- drop-shadow, box-shadow 남발 금지

### 5-3. 보더

| 용도             | 스타일                       |
| ---------------- | ---------------------------- |
| 카드 외곽선      | `1px solid #F0EFED` (선택적) |
| 입력 필드        | `1px solid #E0E0E0`          |
| 입력 필드 포커스 | `1.5px solid #0D9488`        |
| 구분선           | `1px solid #F0EFED`          |
| 선택된 카드      | `1.5px solid #0D9488`        |

- 보더는 얇게 (1~1.5px)
- 구분선보다 여백 우선 — 여백만으로 구분이 되면 구분선 생략

## 6. 컴포넌트 기준값

### 6-1. 버튼

| 타입      | 높이 | radius | 폰트       | 배경        | 텍스트    |
| --------- | ---- | ------ | ---------- | ----------- | --------- |
| Primary   | 52px | 12px   | 16px / 600 | primary     | #FFFFFF   |
| Secondary | 52px | 12px   | 16px / 600 | #F0EFED     | secondary |
| Ghost     | 44px | 8px    | 14px / 400 | transparent | primary   |
| Danger    | 52px | 12px   | 16px / 600 | critical    | #FFFFFF   |

- 하단 고정 CTA: 좌우 20px 패딩, 하단 safe area 확보
- disabled: opacity 0.4, 클릭 불가
- pressed: 배경색 살짝 어둡게 (hover/pressed 파생색 사용)

### 6-2. 카드

- 배경: `#FFFFFF`
- 패딩: `16px`
- 둥글기: `16px`
- 그림자: `shadow-sm` 또는 `border: 1px solid #F0EFED`
- 카드 간 간격: `12px`

### 6-3. 입력 필드

- 높이: `48px`
- 패딩: `0 16px`
- 둥글기: `8px`
- 배경: `#FFFFFF`
- 보더: `1px solid #E0E0E0` → 포커스 시 `1.5px solid #0D9488`
- placeholder: `#9E9E9E`

### 6-4. 칩 (선택지)

- 높이: `40px`
- 패딩: `0 16px`
- 둥글기: `8px`
- 기본: `border: 1px solid #E0E0E0`, 배경 `#FFFFFF`
- 선택: `border: 1.5px solid #0D9488`, 배경 `#E0F2F1`
- 텍스트: `14px / 500`

### 6-5. 프로그레스 바

- 높이: `4px`
- 둥글기: `9999px`
- 트랙: `#F0EFED`
- 채움: `primary`

### 6-6. 뱃지 / 태그

- 패딩: `4px 8px`
- 둥글기: `8px`
- 폰트: `12px / 500`
- 배경/텍스트: 의미에 따라 (tertiary/primary, warning 10%/warning, critical 10%/critical)

### 6-7. 체크리스트 아이템

- 높이: 최소 `56px`
- 구조: 체크박스(24px) + 텍스트 + 우측 메타정보
- 체크박스 미완료: `border: 2px solid #E0E0E0`, 배경 투명
- 체크박스 완료: `배경 primary`, 체크마크 `#FFFFFF`
- 완료 시 텍스트: `line-through`, 색상 `#9E9E9E`

## 7. 아이콘

### 7-1. 라이브러리

- **lucide-react** 사용
- 일관된 스트로크 스타일 (rounded, 1.5px stroke)

### 7-2. 사이즈

| 사이즈 | 값   | 용도                          |
| ------ | ---- | ----------------------------- |
| sm     | 16px | 인라인 아이콘 (텍스트 옆)     |
| md     | 20px | 리스트 아이콘, 버튼 내 아이콘 |
| lg     | 24px | 네비게이션, 탭바, 헤더 액션   |
| xl     | 32px | 빈 상태, 기능 선택 카드       |

### 7-3. 색상 규칙

- 기본: `secondary` (#333344)
- 비활성: `#9E9E9E`
- 강조: `primary` (#0D9488)
- 시스템: 해당 시스템 컬러 (warning, critical, success)
- 아이콘만 단독으로 컬러를 쓰지 않는다 — 텍스트와 함께일 때 동일 색상

### 7-4. 사용 원칙

- 의미 전달 목적일 때만 사용 (장식 금지)
- 아이콘 + 텍스트가 기본. 아이콘만 단독 사용은 네비게이션/액션 버튼에 한정
- 아이콘만 사용할 때는 `aria-label` 필수

## 8. 애니메이션 / 전환

### 8-1. 원칙

- **체감 성능을 높이는 곳에만** 사용한다
- 장식적 애니메이션 없음 — 토스 스타일은 콘텐츠 자체에 집중
- `prefers-reduced-motion: reduce` 시 모든 애니메이션 비활성화

### 8-2. 허용하는 애니메이션

| 대상          | 속성                | duration | easing                         |
| ------------- | ------------------- | -------- | ------------------------------ |
| 버튼 pressed  | background-color    | 100ms    | ease-out                       |
| 페이지 전환   | opacity             | 200ms    | ease-in-out                    |
| 토스트 진입   | transform + opacity | 250ms    | ease-out                       |
| 토스트 퇴장   | opacity             | 150ms    | ease-in                        |
| 바텀시트 진입 | transform (Y)       | 300ms    | cubic-bezier(0.33, 1, 0.68, 1) |
| 프로그레스 바 | width               | 300ms    | ease-out                       |
| 체크박스 토글 | transform (scale)   | 150ms    | ease-out                       |
| 스켈레톤 로딩 | opacity (shimmer)   | 1.5s     | ease-in-out (반복)             |

### 8-3. 금지하는 애니메이션

- 바운스, 스프링 효과
- 화면 전체 슬라이드 전환 (웹뷰에서 네이티브 전환과 충돌)
- 로딩 스피너 외의 무한 반복 애니메이션
- parallax 스크롤
- 요소 등장 시 아래에서 위로 올라오는 stagger 애니메이션

### 8-4. 로딩 상태

- 스켈레톤 UI 우선 (Spinner보다 체감 속도 빠름)
- Spinner는 버튼 내부 로딩, 전체 페이지 초기 로딩에만 허용
- 데이터가 0.3초 이내로 오면 로딩 UI 보여주지 않음 (깜빡임 방지)

## 9. 와이어프레임 대비 변경 사항

기존 와이어프레임(Google Stitch 생성)은 "당근마켓/오늘의집" 따뜻한 스타일로 제작됨.
토스 스타일 전환 시 아래 항목이 변경된다:

| 항목             | 기존 (따뜻한 스타일)         | 변경 (토스 스타일)             |
| ---------------- | ---------------------------- | ------------------------------ |
| 랜딩 일러스트    | 사람+박스 일러스트 크게 배치 | 제거 또는 최소화 (텍스트 중심) |
| 카드 그림자      | 뚜렷한 그림자                | 미세하거나 border로 대체       |
| 버튼 둥글기      | pill형 (9999px)              | 12px radius                    |
| 색상 사용        | tertiary 배경 넓게           | 흰색/회색 위주, primary 아끼기 |
| 온보딩 카드 선택 | 아이콘+라벨 큰 카드          | 간결한 칩 또는 리스트          |
| 전체 톤          | 일러스트+따뜻한 문구         | 깔끔한 텍스트+넉넉한 여백      |
| CheckTip 박스    | 초록 배경 + 캐릭터 아이콘    | 연한 배경 + 텍스트만           |

## 10. Tailwind 매핑

index.css의 `@theme`에 정의된 컬러 토큰과 타이포 토큰을 그대로 사용한다.

```css
@theme {
  /* 컬러 */
  --color-primary: oklch(0.6 0.104 184.7);
  --color-secondary: oklch(0.33 0.03 285);
  --color-tertiary: oklch(0.95 0.019 193);
  --color-neutral: oklch(0.98 0.003 85);
  --color-warning: oklch(0.7 0.187 48);
  --color-critical: oklch(0.64 0.208 25);
  --color-success: oklch(0.7 0.149 163);
  --color-muted: oklch(0.53 0.025 286);
  --color-placeholder: oklch(0.7 0 0);
  --color-border: oklch(0.95 0.003 85);
  --color-border-input: oklch(0.91 0 0);
  --color-surface: oklch(1 0 0);

  /* 타이포그래피 (Tailwind v4: --text-* → text-* 유틸리티) */
  --text-h1: 26px;
  --text-h1--line-height: 1.35;
  --text-h2: 22px;
  --text-h2--line-height: 1.35;
  --text-h3: 18px;
  --text-h3--line-height: 1.4;
  --text-body: 16px;
  --text-body--line-height: 1.6;
  --text-body-sm: 14px;
  --text-body-sm--line-height: 1.5;
  --text-caption: 12px;
  --text-caption--line-height: 1.5;
}
```

### 타이포 토큰 사용법

| 클래스         | 사이즈 | line-height | Weight (별도 지정)    | 용도             |
| -------------- | ------ | ----------- | --------------------- | ---------------- |
| `text-h1`      | 26px   | 1.35        | 700 (`font-bold`)     | 페이지 메인 질문 |
| `text-h2`      | 22px   | 1.35        | 700 (`font-bold`)     | 섹션 제목        |
| `text-h3`      | 18px   | 1.4         | 600 (`font-semibold`) | 카드 제목        |
| `text-body`    | 16px   | 1.6         | 400                   | 본문             |
| `text-body-sm` | 14px   | 1.5         | 400                   | 보조 설명        |
| `text-caption` | 12px   | 1.5         | 400                   | 날짜, 뱃지, 안내 |

> `text-h1 font-bold`처럼 사이즈 토큰 + weight를 조합해서 사용한다.
> `text-[26px] leading-[1.35]` 같은 임의값 사용 금지.

### 자주 쓰는 클래스 조합 예시

```
/* 카드 */
bg-surface rounded-2xl p-4 shadow-sm

/* 페이지 제목 */
text-h1 font-bold tracking-tight text-secondary

/* 보조 텍스트 */
text-body-sm text-muted

/* CTA 버튼 */
h-[52px] w-full rounded-xl bg-primary text-white font-semibold

/* 칩 (미선택) */
h-10 px-4 rounded-lg border border-border-input text-sm

/* 칩 (선택) */
h-10 px-4 rounded-lg border-[1.5px] border-primary bg-tertiary text-primary text-sm font-medium
```

## 부록: 토스 스타일 핵심 요약

토스가 다른 앱과 다른 점을 3가지로 압축하면:

1. **큰 텍스트로 말한다** — 일러스트 대신 26px 볼드 텍스트가 화면 주인공
2. **여백으로 숨 쉰다** — 요소 사이 간격이 넉넉해서 복잡한 정보도 편하게 읽힌다
3. **색은 의미 있을 때만** — 배경은 거의 흰색/회색, 컬러는 "여기 봐" 신호일 때만 등장
